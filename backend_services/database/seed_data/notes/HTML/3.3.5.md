## **1\. Pengenalan Laman Web Interaktif**

* **Definisi:** Laman web yang membenarkan pengguna berinteraksi dengan kandungan, bukan sekadar membaca.  
* **Aktiviti Interaktif:** Termasuk mengisi borang, meninggalkan komen, menekan butang pilihan, dan memberi respons kepada kotak dialog (*popup*).  
* **Bahasa Penskripan:** Menggunakan **JavaScript** (bahasa penskripan klien) untuk memproses interaksi ini.

---

## **2\. Pengesahan Data (*Data Validation*)**

Proses menyemak data yang dimasukkan pengguna dalam borang sebelum ia diproses.

### **A. Semakan Kekosongan Data**

* Memastikan pengguna tidak membiarkan medan penting kosong.  
* **Contoh:** Jika medan nama kosong, kotak amaran (*alert*) akan muncul: "Sila masukkan nama anda".  
* **Kod:** Menggunakan syarat `if (teksnama == "")`.

### **B. Semakan Jenis Data (Nombor)**

* Memastikan pengguna memasukkan jenis data yang betul, contohnya nombor giliran.  
* **Fungsi:** Menggunakan `isNaN()` (*is Not a Number*) untuk mengesan jika input bukan nombor.  
* **Contoh:** Jika huruf dimasukkan dalam medan nombor, amaran akan dipaparkan.

---

## **3\. Butang Pilihan**

Terdapat dua jenis butang utama untuk pengguna membuat pilihan:

### **A. Butang Radio (*Radio Button*)**

* **Fungsi:** Membenarkan pengguna memilih **satu sahaja** pilihan daripada beberapa pilihan (Contoh: Jantina atau Mod Pengajian).  
* **Sifat:**  
  * Menggunakan tag `<input type="radio">`.  
  * Atribut `name` mestilah **sama** bagi semua butang dalam kumpulan itu supaya hanya satu boleh dipilih.  
  * `id` yang unik digunakan untuk rujukan dalam skrip.

### **B. Butang Checkbox**

* **Fungsi:** Membenarkan pengguna memilih **lebih daripada satu** pilihan (Contoh: Subjek yang ingin didaftar).  
* **Sifat:**  
  * Menggunakan tag `<input type="checkbox">`.  
  * Atribut `name` boleh sama untuk kumpulan yang sama.  
  * Pengguna bebas menanda (tick) beberapa kotak serentak.

---

## **4\. Kotak Popup (Kotak Dialog)**

JavaScript menyediakan tiga jenis kotak popup untuk interaksi:

### **A. Kotak Alert**

* **Fungsi:** Memberi amaran atau maklumat ringkas kepada pengguna.  
* **Sintaks:** `window.alert("Mesej anda");`.  
* **Contoh:** Memberitahu pengguna "Anda perlu memasukkan nama anda".

### **B. Kotak Confirm**

* **Fungsi:** Mendapatkan pengesahan daripada pengguna (Ya atau Tidak).  
* **Sintaks:** `window.confirm("Mesej pengesahan");`.  
* **Interaksi:** Mempunyai butang **OK** dan **Cancel**.  
  * Jika **OK**: Mengembalikan nilai `true` (Proses diteruskan).  
  * Jika **Cancel**: Tiada tindakan lanjut.

### **C. Kotak Prompt**

* **Fungsi:** Meminta pengguna memasukkan data (input) sebelum masuk ke laman web.  
* **Sintaks:** `window.prompt("Mesej", "Nilai Default");`.  
* **Interaksi:** Pengguna menaip jawapan dalam kotak dan menekan OK. Data tersebut kemudian boleh dipaparkan dalam laman web (Contoh: "Hai Aminah\!").

---

## **5\. Bahasa Penskripan Pelayan (*Server-side Scripting*)**

* **Definisi:** Digunakan untuk laman web dinamik yang kandungannya berubah mengikut permintaan pengguna (Contoh: Memaparkan biodata murid berdasarkan carian nombor murid).  
* **Ciri-ciri:**  
  * Menggunakan bahasa seperti **PHP** atau **ASP**.  
  * Mempunyai capaian kepada **Pangkalan Data** untuk menyimpan dan mencapai maklumat.  
  * Kandungan sentiasa dikemas kini.
