- Laravel uses MVC model
	- Model: backend services
		- manage data 
		- business logic
	- View: frontend, UI
		- display information to users
	- Controller: middleware, APIs
		- handle requests
		- coordinate responses
---
# Controllers
```
php artisan make:controller <ControllerName>
```
## index()
Convention for displaying list of things

When user navigates for base URI for resource, `index()` is invoked to provide initial view of resource

## Point `web.php` to `index()`
In `web.php`,
```
Use App\Http\Controllers\<ControllerName>;

Route::get('\', [<ControllerName>::class, 'index']);
```
- Breakdown:
	- Importing as a class to pass a particular method
	- Pass `index()` to `web.php`
## view()
- Helper function
- Returns view to user
- Can pass data as into views
```
return view('home', ['chirps' => $chirps]);
```

In view,
```
<h1>{{ $chirps['time'] }}</h1>
```

>PHP Recap
>Can use `foreach($chirps as $chirp)` to iterate through array

# Resource Controllers
- Typically used in standard MVC structure
- Allows for optional model
```
php artisan make:controller <ControllerName> --resource
```

- Methods ready in resource controllers
	- `index()`
		- display list of all resources
	- `create()`
		- showing form to create new object
	- `store()`
		- to create and save new data
	- `show()`
		- display single object
	- `edit()`
		- display form for editing object
	- `update()`
		- to update single object
	- `destroy()`
		- delete chirp
# MVC Flow
When site is visited
1. /route receives request
	1. calls controller, specifically index()
2. Controller prepares data
	1. Pass information to view
3. View renders HTML with data
